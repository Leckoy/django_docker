# Профиль «Информационные технологии»
## Командный кейс № 2 «Управление столовой»

### Описание проекта

Современные школы нуждаются в удобных инструментах для организации питания учащихся. Использование бумажных журналов, талонов и прочих устаревших систем учёта не позволяют достаточно качественно, быстро и эффективно контролировать данный процесс.

Для решения этой задачи было разработано веб-приложение, которое значительно упрощает организацию питания.

### Функциональные возможности

Система разделена на три типа пользователей: ученики, повара и администраторы.

#### Для учеников
- Авторизация и регистрация в системе.
- Просмотр меню завтраков и обедов.
- Оплата заказов (разово или покупка абонемента).
- Указание аллергий в профиле.
- Получение уведомлений о заказе блюда, содержащего аллергенный ингредиент.
- Отметка о получении питания.
- Возможность оставлять отзывы о блюдах.

#### Для поваров
- Авторизация в системе.
- Учёт выданных завтраков и обедов.
- Контроль остатков продуктов и готовых блюд.
- Создание и отправка заявок на закупку продуктов.

#### Для администраторов
- Авторизация в системе.
- Просмотр статистики оплат и посещаемости.
- Согласование заявок на закупку продуктов.
- Формирование отчётов по питанию и затратам.

### Установка и запуск

Для запуска проекта выполните следующие шаги:

1.  **Установить Git:**
    Скачайте и установите Git с [официального сайта](https://git-scm.com/downloads/).
2.  **Установить Python:**
    Скачайте и установите Python с [официального сайта](https://python.org).
3.  **Установить Docker:**
    Скачайте и установите Docker Desktop с [официального сайта](https://docker.com).
4.  **Открыть командную строку** (терминал).
5.  **Перейти в нужную папку:**
    Укажите путь до директории, куда будет клонирован проект.
    ```
    cd <путь_до_вашей_папки>
    ```
6.  **Клонировать репозиторий:**
    ```
    git clone https://github.com/Leckoy/django_docker.git
    ```
7.  **Перейти в папку проекта:**
    ```
    cd django_docker
    ```
8.  **Установить зависимости Python:**
    ```
    pip install -r requirements.txt
    ```
9.  **Синхронизировать базу данных:**
    Выполните миграции для настройки схемы базы данных.
    ```
    python manage.py migrate
    ```
10. **Запустить сервер:**
    Используйте Docker Compose для сборки и запуска контейнеров.
    ```
    docker-compose up
    ```
